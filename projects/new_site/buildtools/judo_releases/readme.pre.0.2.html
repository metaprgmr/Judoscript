Install and Run

Now that you are reading this page, you have successfully obtained the software and have it unpacked. Check the following list before run any JudoScript programs:

  1. Your JVM version is JDK 1.3 or equivalent or highter.
  2. Put file judo.jar in your JVM's class path.
  3. If you intend to use JDBC, make sure the JDBC driver classes for your database is in the class path.
  4. If you intend to use XML features, make sure a Java XML parser is in the class path. We recommend Apache Xerces parser, available from Apache's site. For JDK 1.4 users, a XML parser is already included; Xerces may still be preferrable for advanced XML features. Just make sure Xerces class files are before JDK classes.
  5. If you intend to send e-mail, make sure javax.mail.* classes are in your class path. The JavaMail package can be downloaded from JavaSoft's site. It is also part of J2EE.
  6. If you intend to use JUSP to develop web sites, register the servlet class wws.judo.jusp.JuspServlet to your servlet engine and provide a juspRoot init parameter that serves as the root directory for all JUSP pages. If it is not specified, JUSP pages are resolved the same way as other web pages.

Release Notes

This is an early access release for JudoScript. The Language Specification has listed all the features implemented so far. Some features are listed without details; they are planned for later releases; there is no guarantee, though. Features that are finished or nearly finalized are explained in the articles.

In this release, the language engine and Java gateway are considered completed; language definition may undergo changes based on user feedback but is expected to be minor. Most general-purpose and enterprise functionalities are in. JDBC scripting has been tested mostly for basic database operations; advanced data types and scrollable cursors are not supported as yet. XML scripting supports SAX and DOM programming; other XML processing activities can be achieved by using Java.

Since this is a rather early access release, some obvious language features are not in yet, such as precompiled programs or even byte-code generation. Error handling is fairy rudimental, that is, parsing errors and runtime errors may look very different because the former is generated by JavaCC. This will be customized later. Line numbers reported in included files may not be accurate. All these will be collectively addressed in error handling clean-up.

JudoScript has so many features that make it a legitimate shell. The current release is more of a language engine; commands will be introduced along with a shell facility. The language engine can be customized by a startup script. These are part of a to-do list.

Changes

The following is a list of changes in addition to internal bug fixes and enhancements:

  1. Bean Scripting Framework support.
  2. Bug fix: XML packages are mandatory, not optional.

Documentation and Examples

Documentation includes a language specification that describes each feature precisely and a set of articles discussing major features with a lot of code examples. The source code of the formal code listings are all included in the examples directory.

Report Bugs and Feedbacks

To report bugs, submit feedback or have a question, come to www.judoscript.com and click the feedback link. When submitting bugs, check the latest version first, and make sure the version number is included in the message. To get a version information, do this: 

  java judo -x println versionInfo()
